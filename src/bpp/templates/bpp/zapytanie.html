    Skladnia DjangoQL — rozwin pomoc i przyklady

    Zapytania pisz w postaci pole operator wartosc, laczone przez and / or. Stringi obejmij cudzyslowem. Edytor podpowiada nazwy pol i wartosci — uzyj klawiszy ↑ ↓ do wyboru, Tab aby wstawic.

    OperatorZnaczenie
    =, !=rowne / rozne
    >, >=, <, <=porownania liczbowe / dat
    ~, !~zawiera / nie zawiera (tekst)
    startswith, endswithzaczyna sie / konczy
    in, not innalezy do listy: rok in (2023, 2024)

    Przyklady — rozwin wg poziomu trudnosci. Kliknij wybrane zapytanie, zeby je wstawic do edytora.

    {% for level in examples %}
    {{ level.level }}. {{ level.title }} — {{ level.subtitle }} {% for group in level.groups %}

    {{ group.label }}:

      {% for desc, query in group.items %}
    • {% if desc %}{{ desc }}:{% endif %} {{ query }}
      • {% endfor %}
    {% endfor %}
    {% endfor %}
    Wskazowki dla power-userow
    • Grupowanie — nawiasy do logicznych grup: (A or B) and C.
    • Negacja — not przed wyrazeniem: not (charakter_formalny.skrot = "AC").
    • Pola relacji — po kropce, mozna zagniezdzac: charakter_formalny.skrot, zrodlo.wydawca.nazwa.
    • Listy wartosci — in (...) oszczedza wielu or.
    • Daty / datetime — ISO-string w cudzyslowie: "2025-01-01".
    • Pusty FK — pole != None / pole = None; dla stringa pole = "".
    • Granice zakresow — brak between, zlozyc z >= X and <= Y.
