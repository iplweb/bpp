Każdy build obrazu Docker generuje teraz Software Bill of Materials (CycloneDX 1.6) — ``cyclonedx-py environment`` skanuje produkcyjny venv po usunięciu zbędnych pakietów (matplotlib z pygad). SBOM jest shipowany w obrazie pod ``/app/sbom.json`` i uploadowany jako workflow artifact (90-dniowa retencja). Pozwala szybko sprawdzić czy działający obraz zawiera podatne wersje pakietów gdy nowy CVE jest disclosed (``docker run --rm <image> cat /app/sbom.json | jq '.components[].name'``).
